Task Descriptor: Planejar o Projeto
Uma tarefa colaborativa que descreve um acordo inicial sobre como o projeto vai entregar o Visão do produto. O plano de projeto resultante fornece uma visão geral resumida do projeto.

Main Description
Desenvolver o plano de projeto constitui uma oportunidade para obter consenso da equipe sobre o escopo do projeto, os objetivos, o cronograma inicial e os itens a serem entregues. Ele permite à equipe começar a demonstrar auto-organização, definindo os critérios de sucesso e práticas de trabalho a serem utilizados. A meta é a colaboração e o consenso de todos os principais participantes do projeto, mas o gerente de projeto tem a responsabilidade final de garantir que todos estejam comprometidos com o plano.
Steps
Estabeleça uma equipe coesa
O planejamento do projeto, mesmo ao nível de resumo, não deve ser feito de forma isolada, visto que ele descreve o que o projeto irá entregar e como. A equipe começa discutindo quem desempenha quais papéis e concorda com as suas responsabilidades. O gerente de projeto precisa certificar-se de que a alocação das pessoas esta sendo feita de acordo com os interesses do projeto e que cada papel necessário esteja direcionado.
Determine o tamanho e o escopo do projeto

A equipe produz estimativas aproximadas de duração para cada item na Artifact: Lista de Itens de Trabalho (veja Guideline: Estimativa Ágil). Considerando a Visão do produto, as estimativas serão utilizadas para discutir com os Stakeholders, de forma realista, as prioridades do que deve ser desenvolvido, atendendo as restrições do projeto.

Se o projeto for orientado a características, a equipe deve verificar quantas pessoas são necessárias para terminar estes itens de trabalho, o que lhes dará uma vaga compreensão da duração do projeto, do perfil dos funcionários e do escopo. Se ao contrário, o projeto for orientado a datas, a equipe estima quanto trabalho pode ser feito no tempo definido, usando a equipe disponível. O trabalho que esteja fora do escopo pode ser considerado nas liberações futuras.
Avalie os riscos

A equipe identifica os riscos do projeto, executa uma análise qualitativa dos riscos para avaliar a sua magnitude e atualiza a Artifact: Lista de Riscos. O gerente de projeto facilita a decisão da equipe sobre quais riscos eles deverão resolver, e quais eles deverão só observar.

A solução pode ser evitar ou atenuar os riscos, explorando as oportunidades ou aumentando a probabilidade e os impactos positivos do risco. Dependendo do caso, alguns itens de trabalho podem ser adicionados ou removidos da Lista de Itens de Trabalho para certificar que as soluções serão priorizadas e tratadas pela equipe, junto com o trabalho no projeto. Como não é possível planejar soluções para todos os riscos identificados, a equipe pode decidir aceitar alguns deles. Os riscos a serem observados serão comunicados aos Stakeholders e permanecerão na Lista de Riscos e as ações de correção serão determinadas apenas se eles ocorrerem.
Descreva o ciclo de vida do projeto

Defina a duração da iteração e utilize-a para avaliar a velocidade (veja Guideline: Estimativa Ágil). A quantidade de itens a serem entregues em cada iteração será definida pela velocidade da equipe e pelas estimativas para cada item. A equipe usa a Lista de Itens de Trabalho para descrever quais características serão implementadas em quais iterações, colocando os itens de trabalho de maior prioridade por primeiro, incluindo respostas planejadas para os maiores riscos ou oportunidades.

O OpenUP organiza as iterações em um conjunto de fases. Cada fase no ciclo de vida do projeto irá terminar com um marco destinado a fornecer aos Stakeholders mecanismos de supervisão e direção para controlar no projeto, o financiamento, o escopo, a exposição a riscos, o valor fornecido e outros aspectos do processo (veja Concept: Ciclo de Vida de Projeto).

Você não precisa gastar muito tempo fazendo este planejamento. O Plano do Projeto deve documentar apenas um breve resumo dos marcos do projeto e de um a três objetivos para cada iteração. Não coloque itens de trabalho individuais no plano, pois isto forçará muito o re-planejamento. A meta é criar um plano genérico que esboce como a equipe poderá construir a aplicação nas iterações definidas. Níveis extras de detalhes serão realizados em outras sessões de planejamento ao longo do projeto (veja Task: Planejar a Iteração). Pode ser necessário que você revise este plano mais tarde para adaptá-lo com base no que você irá aprender executando as iterações (veja os Artifact: Plano de Iteração das iterações anteriores).
Estabeleça os custos e articule o valor

Desenvolva uma estimativa de ordem de grandeza para os custos dos recursos necessários à conclusão dos itens de trabalhos do projeto. Um modelo simples para custeio do projeto pode ser a multiplicação do custo médio da equipe pela duração de uma iteração, derivando o impacto financeiro em curso (ou seja, o custo por iteração). Esta primeira fase de planejamento deve manter as coisas muito genéricas e flexíveis. A meta é a articulação do valor versus as restrições orçamentárias do projeto e a ajuda aos Stakeholders para que decidam se é oportuno prosseguir com o projeto ou não. Se for necessário proponha alternativas para diminuir os custos, tal como a remoção, do escopo do projeto, de itens de trabalho de baixo valor e alto custo.
Planeje a implantação
Planeje a estratégia para implantar o software (e suas atualizações) no ambiente de produção. Discuta o cronograma da liberação com os departamentos de operações e suporte para assegurar que o projeto se enquadra no sistema corporativo de implantação. Se você estiver substituindo um sistema existente, decida se irá executar o novo sistema, em paralelo com ele ou se você vai realizar uma troca imediata. Você também poderá ter que negociar com os proprietários dos sistemas os quais o novo sistema tem dependências. Atualize a Lista de Itens de Trabalho com o trabalho adicional que possa ser necessário para a implantação. Adicione riscos de implantação significativos na Lista de Riscos e, se necessário, faça ajustes no Plano do Projeto.
